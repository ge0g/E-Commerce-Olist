# Ноутбуки проекта

В этой папке находятся Jupyter Notebook-файлы, которые последовательно показывают ход анализа данных Olist: от первичного обзора исходных таблиц до подготовки аналитической базы и бизнес-анализа.

Ноутбуки рекомендуется запускать по порядку.

---

## 1. `01_data_overview.ipynb`

Первичный обзор исходных данных.

В ноутбуке проверяются:

- наличие файлов;
- размеры таблиц;
- структура столбцов;
- типы данных;
- пропуски;
- дубликаты;
- уникальность ключей;
- статусы заказов;
- диапазон дат;
- связи между таблицами.

Цель ноутбука — понять структуру и качество исходных данных перед подготовкой аналитической базы.

---

## 2. `02_data_preparation.ipynb`

Подготовка аналитических таблиц для дальнейшего анализа.

В ноутбуке выполняются:

- преобразование дат;
- объединение таблиц;
- подготовка справочника товаров и категорий;
- агрегация товарных позиций до уровня заказа;
- агрегация оплат до уровня заказа;
- агрегация отзывов до уровня заказа;
- формирование основной аналитической базы заказов;
- выделение валидных продаж;
- расчёт признаков доставки;
- определение новых и повторных клиентов;
- клиентская сегментация;
- создание таблиц для анализа категорий и доставки;
- сохранение подготовленных таблиц в `data/processed`.

Клиентская сегментация в проекте не является классическим RFM-анализом. Вместо этого используется адаптированная бизнес-сегментация по трём признакам:

- общей сумме покупок клиента;
- давности последней покупки;
- факту повторных заказов.

Такой подход выбран из-за низкой повторности клиентов в датасете Olist: большинство клиентов сделали только один заказ, поэтому классический показатель частоты покупок плохо разделяет клиентов.

---

## 3. `03_business_analysis.ipynb`

Бизнес-анализ подготовленных данных.

В ноутбуке рассматриваются пять основных бизнес-вопросов:

1. Как менялись выручка, количество заказов и средний чек Olist по месяцам?
2. Какие категории товаров вносят наибольший вклад в выручку и количество заказов?
3. Какую роль в продажах играют новые и повторные клиенты?
4. Какие клиентские сегменты наиболее ценны для бизнеса?
5. Как сроки и задержки доставки связаны с оценками клиентов?

Основной период анализа: **2017-01 — 2018-07**.

Период зафиксирован явно, чтобы исключить неполные и нестабильные месяцы в начале и конце датасета и не искажать динамику продаж.

Финальные графики из этого ноутбука сохраняются в папку:

```text
reports/figures/
```

---

## Порядок запуска

1. Поместить исходные CSV-файлы в папку:

```text
data/raw/
```

2. Запустить ноутбук первичного обзора:

```text
notebooks/01_data_overview.ipynb
```

3. Запустить ноутбук подготовки данных:

```text
notebooks/02_data_preparation.ipynb
```

4. Запустить ноутбук бизнес-анализа:

```text
notebooks/03_business_analysis.ipynb
```

После запуска `02_data_preparation.ipynb` в папке `data/processed` будут созданы подготовленные CSV-файлы, необходимые для бизнес-анализа и SQL-проверки.

---

## Важные замечания

- Сырые данные из `data/raw` не хранятся в репозитории.
- Подготовленные CSV-файлы из `data/processed` не хранятся в репозитории.
- Для воспроизведения анализа исходные данные нужно загрузить локально в `data/raw`.
- Основной бизнес-анализ находится в `03_business_analysis.ipynb`.
- Подробные итоговые выводы и бизнес-рекомендации вынесены в основной `README.md`.
